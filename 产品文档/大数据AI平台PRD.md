# 大数据AI平台 — 模型中心 PRD

## 1. 产品概述

### 1.1 产品定位

大数据AI平台是面向集团内部的AI模型统一管理与监控平台，核心能力包括模型状态实时监控、调用量与成功率趋势分析、异常快速定位。平台将分散在AI巡店、AI巡仓、SKU识别等业务线下的模型进行集中化治理，帮助运营与技术人员掌握模型运行全貌。

### 1.2 目标用户

| 角色 | 核心诉求 |
|------|---------|
| 运营人员 | 快速了解模型是否正常运行、调用量是否达标 |
| 技术运维 | 定位异常模型、分析成功率波动原因 |
| 管理层 | 全局掌握AI模型资产与运行健康度 |

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Vue 3 + TypeScript |
| 构建 | Vite 5 |
| UI组件库 | Element Plus |
| 状态管理 | Pinia |
| 图表 | ECharts |
| 路由 | Vue Router 4 |

## 3. 功能详细说明

### 3.1 模型中心

**路径**: `/model-center`  
**功能**: 展示系统中所有模型，支持多维度筛选与分页浏览

#### 3.1.1 筛选条件

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 模型分类 | 下拉单选 | 可选：AI巡店、AI巡仓、SKU识别；默认全部 |
| 模型名称 | 文本输入 | 支持模糊匹配，可清空 |
| 模型状态 | 下拉单选 | 可选：正常、空闲、异常；默认全部 |
| 时间 | 日期范围选择器 | 默认当天 |

**操作按钮**：
- 查询：按筛选条件过滤，重置到第1页
- 重置：清空所有筛选条件，恢复默认值

#### 3.1.2 卡片展示

- **布局**：每行4张卡片（基于24栅格，每卡span=6，间距20px）
- **卡片内容**：

| 字段 | 说明 |
|------|------|
| 模型名称 | 卡片顶部，粗体，最多2行截断 |
| 分类标签 | 模型所属业务线名称 |
| 状态标签 | 正常（绿色）、空闲（橙色）、异常（红色） |
| 调用量 | 右下角，千分位格式化 |

- **交互**：点击卡片跳转至模型详情页
- **Hover效果**：卡片上浮2px + 蓝色边框 + 阴影增强

#### 3.1.3 分页

| 选项 | 说明 |
|------|------|
| 20条/页 | 默认值 |
| 80条/页 | - |
| 全部 | 不分页，展示所有筛选结果 |

- 显示总条数、页码跳转

#### 3.1.4 模型分类预设

| 分类ID | 分类名称 | 模型数量 | 模型清单 |
|--------|---------|---------|---------|
| 1 | AI巡店 | 7 | 陈列饱满度模型、T形取货模型、商品陈列识别模型、货架缺货检测模型、价格标签识别模型、促销堆头识别模型、门店卫生检测模型 |
| 2 | AI巡仓 | 6 | 仓库温湿度监测模型、库存异常检测模型、货物摆放规范模型、仓库安全监测模型、入库质检模型、出库复核模型 |
| 3 | SKU识别 | 7 | SKU条码识别模型、SKU图像识别模型、商品分类识别模型、保质期识别模型、商品规格识别模型、新品录入识别模型、货架SKU匹配模型 |

### 3.2 模型详情监控

**路径**: `/model-detail/:id`  
**功能**: 单个模型的实时状态监控与调用量趋势分析

#### 3.2.1 面包屑导航

```
模型中心 / {模型名称}
```

点击「模型中心」返回列表页。

#### 3.2.2 模型基本信息卡片

| 字段 | 示例 |
|------|------|
| 模型名称 | 陈列饱满度模型 |
| 分类 | AI巡店 |
| 版本 | V1.0 |
| 创建者 | 张三 |
| 创建时间 | 2026.04.30 |

#### 3.2.3 状态指标卡片（三栏等高对齐）

| 卡片 | 内容 | 说明 |
|------|------|------|
| 近10分钟状态 | 状态标签 | 正常/空闲/异常，对应绿/橙/红色圆角标签 |
| 近10分钟调用成功率 | 百分比数值 | 点击可跳转成功率详情页；<95%显示红色，≥95%显示绿色；下方标注更新时间 |
| 调用量 / 调用异常次数 | 两个数值 | 蓝色（调用量）/ 橙色（异常次数）大号字体 |

**状态判定规则**：

| 状态 | 判定条件 |
|------|---------|
| 空闲 | 最近一个10分钟时段无调用 |
| 异常 | 近10分钟调用成功率 < 95% |
| 正常 | 其余情况（有调用且成功率 ≥ 95%） |

**刷新机制**：状态和成功率每10分钟自动刷新一次。

#### 3.2.4 调用量趋势折线图

**时间筛选**：
- 类型：日期时间范围选择器
- 粒度：仅允许10分钟间隔（分钟选择器禁用非0/10/20/30/40/50的选项）
- 默认：当天 00:00 至 23:50

**图表规格**：

| 项目 | 说明 |
|------|------|
| 图表类型 | 双Y轴折线图 |
| 左侧Y轴 | 调用量 |
| 右侧Y轴 | 异常次数 |
| X轴 | 固定10分钟时间段标签 |
| 蓝色折线 | 调用量（含渐变面积填充） |
| 橙色折线 | 异常数（含渐变面积填充） |
| Tooltip | 悬停显示：时间段、调用量、异常数、成功率 |

**图例**：调用量（蓝色）、异常数（橙色）

## 4. 数据模型

### 4.1 核心实体

```
ModelInfo {
  id: number               // 模型唯一标识
  name: string             // 模型名称
  category: string         // 分类名称
  categoryId: number       // 分类ID
  status: ModelStatus      // 当前状态：正常 | 空闲 | 异常
  invocations: number      // 累计调用量
  successRate: number      // 近10分钟调用成功率(%)
  errorCount: number       // 近10分钟异常次数
  version: string          // 模型版本号
  creator: string          // 创建者
  createdAt: string        // 创建时间
}

TimeSlotData {
  time: string             // 时间段 "15:30 - 15:40"
  timeLabel: string        // X轴标签 "15:30"
  invocations: number      // 该时段调用量
  errorCount: number       // 该时段异常次数
  successRate: number      // 该时段成功率
  status: ModelStatus      // 该时段状态
}

SuccessRateRecord {
  id: number               // 记录ID
  time: string             // 时间段
  invocations: number      // 调用次数
  errorCount: number       // 异常次数
  successRate: string      // 成功率（空闲显示"-"）
  status: ModelStatus      // 状态
}
```

### 4.2 枚举值

**ModelStatus**：

| 值 | 含义 | 颜色 | 背景色 |
|----|------|------|--------|
| 正常 | 有调用且成功率≥95% | #67c23a | #f0f9eb |
| 空闲 | 近10分钟无调用 | #e6a23c | #fdf6ec |
| 异常 | 成功率<95% | #f56c6c | #fef0f0 |

## 5. 交互流程

### 5.1 模型浏览与监控流程

```
用户登录 → 模型中心（默认展示所有模型）
    ↓ 筛选/翻页
找到目标模型 → 点击卡片 → 模型详情页
    ↓ 查看状态与趋势图
    ├→ 调整时间范围 → 折线图刷新
    └→ 点击成功率数值 → 成功率详情页
        ↓ 按日期/时间段查询
        查看表格化成功率明细
```

### 5.2 状态刷新流程

```
每10分钟触发一次：
  获取模型最新调用数据
  → 计算近10分钟调用量
  → 若调用量=0 → 状态=空闲
  → 若成功率<95% → 状态=异常
  → 否则 → 状态=正常
  → 更新卡片/详情页状态展示
```
